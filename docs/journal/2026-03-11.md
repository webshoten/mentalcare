- スマホでの音声テスト方法を検討
  - HTTP（`http://192.168.x.x:4321`）では `getUserMedia` がブロックされマイク使用不可（Secure Context 必須）
  - cloudflared tunnel を試行 → Vite の `allowedHosts` 問題でページが届かず断念
  - `allowedHosts: true` を astro.config.mjs に追加するもリンターが戻す
- Chime SDK テストページ（音声参加専用）の案を検討中
  - URLパラメータに meetingId + attendeeToken を入れるだけで音声参加できる静的HTMLページ
  - S3 + CloudFront にデプロイすれば HTTPS 自動付与
  - PC側でミーティング作成 → token をURLに含めてスマホに共有 → スマホから音声参加
  - 本体アプリとは独立した軽量テストツールとして使える
  - 実装は HTML 1ファイル + Chime SDK JS のみ
